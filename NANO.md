# Nano CC — 减法分支

> 理解即删除：每拆解一个模块，就精简一份代码

---

## 分支理念

```
原项目: 功能完整的 Claude Code 镜像
   ↓
Nano CC: 精简核心，理解本质
```

**目标**：通过深度理解代码来精简它，而不是添加功能。

---

## 减法原则

1. **先理解，后删除** — 只有完全读懂的代码才能删除
2. **保留核心** — 运行时、路由、注册表是骨架
3. **删除占位** — 25 个空目录是理解后的第一批目标
4. **合并重复** — 功能相似的模块合并
5. **移除 shim** — 模拟执行改为直接返回

```
src/
├── main.py          # CLI 入口
```
